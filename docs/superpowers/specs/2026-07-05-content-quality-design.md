# 内容质量 6 件套设计

日期:2026-07-05
状态:Auto Mode 下自主推进

## 目标

让抓下来的东西更接近原文,同时不牺牲现有的正确性/安全性。

## 决策记录

### 1. 元数据(发布时间/公众号名)

- `data["date"]`:从 `var ct = "unix时间戳"` 解析(实测:公众号 2026 起把 publish_time 从 DOM 里去掉了)
- `data["account"]`:等于现有 `author` 字段(公众号里 `id="js_name"` 是账号名,不是文章作者)。字段名保留 `author` 不改,只是意义澄清
- **阅读量不做**:公众号通过异步 JS 拉取,静态 HTML 拿不到,YAGNI
- HTML 生成:meta 区多显示一行 `2026-06-24`
- Markdown:开头加一行 `> 发布于 2026-06-24 · 公众号:xxx`

### 2. 表格保留

- `<table>` → Markdown 表格 `| a | b |\n|---|---|\n| c | d |`
- 首行(`<thead>` 或第一个 `<tr>`)当表头
- 合并单元格(`colspan`/`rowspan`)简化处理:文字并入第一格
- 加入 `_STRUCTURE_TAGS`,含图/嵌套结构时回退递归
- HTML 输出:标准 `<table>` + 斑马纹 + 边框 CSS

### 3. 图片下载重试

- 当前:`download_image` 一次失败就 `None`
- 改为:最多 3 次(1s / 2s 指数退避)
- 只重试网络类异常(`requests.RequestException`),4xx/5xx 直接放弃
- 保持并发不变(线程池)

### 4. 假标题识别

保守规则,避免误伤:

- `<section>` 或 `<p>` 直接子节点只包含"短文本"(<40 字)
- **且** style 里含 `font-weight:bold`/`700`/`800` 或有 `<strong>` 独占子标签
- **且** 内部无 `<img>`/其他结构标签
- 满足 → 当作 `<h3>` 处理(默认三级,过于自由的字号推断易错)

只在 `_collect_wechat_content` 里生效,不改变已有真标签(h1-h6)的行为。

### 5. 斜体/下划线/删除线

- `<em>/<i>` → `*文字*`
- `<del>/<s>` → `~~文字~~`
- `<u>` → markdown 无原生语法,用 `<u>文字</u>` 直通(需要 generate_html 保留而不转义)

**处理下划线直通的方案**:
在 XSS 转义之前,用占位符法把 `<u>...</u>` 和 `</u>` 保护起来(和代码块类似的机制),转义完再放回。

### 6. 嵌套列表层级

- 当前:`ul/ol` 拍平成一级
- 改为:递归遍历 li,子列表加两空格缩进(标准 markdown)
- 输出示例:

```
- 一级项
  - 二级子项
  - 二级子项
- 一级项
```

- generate_html 的列表正则要允许缩进 `- ` 或 `  - `

阅读量、评论区、多合并单元格精细渲染、下划线用 markdown 扩展语法 `__…__`(和加粗冲突)、字号→标题级别自动映射。
