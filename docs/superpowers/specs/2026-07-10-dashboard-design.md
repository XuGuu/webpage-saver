# 抓取统计仪表盘设计

日期:2026-07-10
状态:Auto Mode 下用户已批准,直接实施

## 目标

生成 `dashboard.html`,一眼看到自己的抓取全貌——像 Spotify Wrapped 那种小仪表盘,而不是简单一行统计。

## 决策记录

### 数据源

- 复用 `scan_saved_articles`(轻量:title/author/date/mtime/html_path/folder)
- 新增 `_read_article_extra_stats(html_path)`:读 HTML 补 `site`(公众号/B 站…)、`word_count`、`image_count`
- `build_dashboard_html(root)` 内部对每篇跑一次 stats,聚合后渲染

### 时间维度选择

- **月度柱状图 / 热力图**:按 **mtime(保存时间)**——反映"你什么时候抓的",而不是文章何时发布
- **最新/最老**:同上,按 mtime
- 数据里如果 date 缺失也不影响(mtime 一定存在)

### 页面结构(从上到下)

```
┌─────────────────────────────────────┐
│   我的抓取仪表盘                     │
├─────────────────────────────────────┤
│ 128 篇 · 24 来源 · 42.6 万字         │
│ · 7,344 张图 · 覆盖 176 天           │
├─────────────┬───────────────────────┤
│ 月度柱状     │ 网站类型分布           │
│ (近 12 月)   │ (conic-gradient 环形)  │
├─────────────┴───────────────────────┤
│ 来源排行 Top 15(横条形图)          │
├─────────────────────────────────────┤
│ 365 天保存热力图(GitHub 风格)      │
├─────────────────────────────────────┤
│ 极值:最长 / 最短 / 最新 / 最老        │
└─────────────────────────────────────┘
```

### 图表实现(全 CSS + 少量内联 JS,不引外部库)

- **核心数字条**:大字号,横排,每个数字 + 标签
- **月度柱状图**:CSS flexbox,每根 `<div>` 高度 = 值 / 最大值 × 200px
- **网站类型分布**:`background: conic-gradient(from 0deg, #a 0% 60%, #b 60% 80%, ...)`——一个 div 就是一个环形图,零 SVG
- **来源排行**:每行一个 `<div>`,内部 `<div>` width: 百分比,显示颜色条
- **热力图**:365 个 grid item,颜色分 5 级(空 / 少 / 中 / 多 / 最多),`grid-template-columns: repeat(53, 1fr)`(53 周)、`grid-auto-flow: column`(按列填,7 行代表周一到周日)

### 极值链接

- 4 张小卡:最长文章 / 最短 / 最新 / 最老,可点击跳转对应 HTML

### 深色模式

跟随系统,和现有 index/generate_html 一致的调色板。

### 集成

- 保存成功后自动更新 `dashboard.html`(和 `目录.html` 并列)
- GUI 加「📊 统计」按钮,点击后:如果不存在就生成一次,然后 `open_file`
- 空数据:显示"还没保存过文章"

## 测试(可自动化)

- `_read_article_extra_stats`:合成 fixture HTML 断言 word_count、image_count、site
- `build_dashboard_html`:空数据 → 含"还没保存"文字;3 篇文章 → 含"共 3 篇"和作者名;热力图 grid 出现 353+个 cell(365 或按周对齐)
- 现有 130 个测试不许坏

## 明确不做(YAGNI)

- 时间范围切换(默认全部;要"最近 30 天"用户可自己筛);
- 导出统计到 CSV;
- 阅读时长估算;
- 自然语言总结("你今年最爱看的作者是……");
- 交互式过滤(点击某作者只显示他的文章)。

## 澄清:排行榜按"作者/账号",不是"网站类型"

设计初稿里的"来源排行 Top 15"中的"来源"实际指**作者/公众号名等具体来源账号**(比如"烨笙总Yes"、"阑夕"),不是"网站类型"(公众号 / 掘金 / B 站…)。
- **网站类型分布**由页面右上角的环形图(conic-gradient)呈现
- **作者/账号排行**是页面中段横条形图,展示按作者数量排序的 Top 15
两者互补,共同回答"最常看谁"和"最常从哪个平台读"两个问题。
