# 知识库化批次(4 项)设计

日期:2026-07-07
状态:Auto Mode 下用户已批准整批,直接实施

## 目标

让工具从"抓文章"进化到"个人小知识库":搜得到、看得到分布、回得去、错得明白。

## 决策记录

### #1 目录索引加搜索框

- build_index_html 顶部加 `<input type="search" placeholder="搜索标题/作者">`
- 内联 JS 监听 input 事件,过滤 `.card` 显示(匹配 title / card-meta 任一子串,不区分大小写)
- 无外部依赖,浏览器原生 JS
- 支持中英文关键字

### #2 抓取统计条

build_index_html h1 下方加"统计条",显示:

- 总数:`共 42 篇`
- 时间跨度:`2026-05-01 → 2026-07-06`
- 来源分布:Top 3 来源加数量,如 `烨笙总Yes 12 · 阑夕 8 · 少数派 5`

数据来源:扫描 saved-article 目录时收集元数据后聚合。空时不显示(0 篇的情况)。

### #3 GUI 最近保存历史列表

- 加按钮「📚 最近保存」
- 点击弹出 `tk.Toplevel` 小窗(400×500),标题「最近保存」
- 列表显示最近 20 篇:标题(粗)、作者 · 日期(灰)
- 双击列表项 → open_file 打开对应 HTML
- 空时显示"还没有保存过文章"
- 数据来源:抽出 `scan_saved_articles(root_dir) -> list[dict]` 复用扫描逻辑

### #4 CSDN 登录墙友好提示

- extract_csdn 里所有降级方案后如内容短(<200 字)且含"登录"字样,抛出增强错误消息:
- 消息模板:
  ```
  CSDN 文章需要登录才能看全文。
  解决办法:
  1. 用浏览器打开该文章
  2. 登录 CSDN 账号(免费)
  3. 回来重新点「开始保存」
  ```
- GUI 显示错误消息用 `messagebox.showinfo`(比 showerror 温和)——只在 CSDN 类错误时用,别的照旧 showerror。或者更简单:错误消息本身带指引即可,不改 UI 层。选后者。

## 测试

每项一组单测(能自动化的):
- 搜索:检查 build_index_html 输出含 `<input type="search">` 和相关 JS
- 统计条:含"共 X 篇"、"→"、"Top" 之类文字标记
- scan_saved_articles:返回结构化数据 + 只识别标记目录
- CSDN 错误消息:mock 短内容 + "登录" 字样,断言异常里含"登录 CSDN"字样

现有 94 个测试不许坏。

## 明确不做(YAGNI)

- 搜索里的高亮命中字符;全文搜索(只搜元数据);统计里的月度柱状图;历史列表分页;CSDN 自动登录(违反用户隐私预期)。
