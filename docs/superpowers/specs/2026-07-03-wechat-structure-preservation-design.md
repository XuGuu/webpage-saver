# 公众号文章结构保留(标题/列表/引用/加粗)设计

日期:2026-07-03
状态:已获用户批准(方案 A)

## 问题

公众号提取器把正文里所有没有图片的盒子统一 `get_text()` 拍平,导致:
- `<h1>`~`<h6>` 标题变普通段落(用户实测截图确认:《Superpowers不再是最佳实践》一文 9 个 `<h2>` 小标题全部丢失层级)
- `<ul>`/`<ol>` 列表失去条目标记
- `<blockquote>` 引用块失去样式
- `<strong>`/`<b>` 加粗失去强调

## 方案选择

- **A(选定)**:增强自研递归收集器 `_collect_wechat_content`,逐结构加识别分支
- B(否决):markdownify 整体转换——新依赖 + 公众号嵌套 HTML 噪音大 + 图片 data-src/编号逻辑要重做
- C(否决):trafilatura——当初就是因为它丢公众号图片位置才自研的

## 设计

### 1. 收集器(save_webpage.py `_collect_wechat_content`)

新增分支(优先级从上到下,插在 img 分支之后):

| 遇到 | 输出 |
|------|------|
| `h1`~`h6`(不含图) | `#`×级别 + 空格 + 文字,h4 以下按 4 级封顶 |
| `ul`/`ol`(不含图) | 每个直接子 `li` 输出 `- 内容` / `n. 内容`,嵌套列表拍平 |
| `blockquote`(不含图) | 每行加 `> ` 前缀 |
| 上述结构含图片时 | 回退为递归深入(不丢图优先于保结构) |
| 普通段落 | 改用行内感知渲染 `_inline_md`:`strong`/`b` 包成 `**…**` |

递归规则:盒子子树含 `img` **或 h1-h6/ul/ol/blockquote** 才递归,否则按段落渲染。
(解决"标题藏在无图嵌套盒子深处被 get_text 拍平"的问题)

新增辅助函数 `_inline_md(el)`:按序拼接行内文本,`strong`/`b` → `**…**`,跳过 script/style/注释。

### 2. HTML 生成器(`generate_html`)

Markdown → HTML 规则表补充:
- `#### ` → `<h4>`
- 连续 `- ` 行 → `<ul><li>…</li></ul>`;连续 `n. ` 行 → `<ol>`
- 连续 `> ` 行 → `<blockquote>`
- `**` → `<strong>` 已有,不动

CSS 补 h4 / ul / ol / li / blockquote 样式,与现有风格一致。

### 3. 测试

- 每种结构一组单元测试(标题各级、深嵌套标题、ul/ol、引用、行内加粗、结构含图回退)
- generate_html 的新转换规则测试
- 现有 10 个测试不许坏
- 真实文章端到端:**手动验证**(不写成自动化测试,保持测试套件离线可跑)。
  2026-07-03 已验证:9/9 个 `## ` 标题恢复、12 处加粗、6/6 图片位置保留。

### 3.1 评审后的修正(2026-07-03 code review)

- `_inline_md` 改为折叠空白为单个空格(而非逐段剥除),保住英文加粗前后的单词边界
- 嵌套加粗(`strong` 套 `b`)不再重复包 `**`
- `generate_html` 末尾增加"解包"后处理:块级元素(h1-h4/ul/ol/blockquote)不再嵌在 `<p>` 里,块后多余 `<br>` 清除
- 已知且接受:以 `- `/`数字. `/`> ` 开头的普通段落会按 Markdown 标准语义转为列表/引用(与 CommonMark 一致)

### 4. 明确不做(YAGNI)

表格、代码块语法高亮、链接内嵌样式、嵌套列表层级。公众号少见,遇到再说。
