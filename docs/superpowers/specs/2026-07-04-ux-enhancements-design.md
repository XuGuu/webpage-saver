# 用户体验增强 5 件套设计

日期:2026-07-04
状态:Auto Mode 下自主推进,用户随时可打断/修改

## 目标

一次实现 5 项 UX 增强,让工具从"能用"进化到"顺手":
1. 保留超链接
2. 保留代码块(含行内 code)
3. macOS 双击启动
4. GUI 打开时自动填充剪贴板中的 URL
5. GUI 支持一次粘多条 URL 批量保存

## 决策记录(选项 + 选定 + 原因)

### 1. 保留超链接

- **选定**:在 `_inline_md` 和 `_collect_wechat_content` 里识别 `<a href>`,输出 `[文字](链接)` 语法
- 原因:generate_html 里已有的 `\[…\]\(…\)` → `<a>` 转换会自动把它变回可点击 HTML;链接过滤:跳过锚点(`#xxx`)和 `javascript:`
- 已知不做:懒加载链接(公众号无此机制)

### 2. 保留代码块

- **选定**:
  - `<pre>` 块(公众号的代码块) → 输出 ```` ```\n内容\n``` ````
  - 行内 `<code>` → 输出 `` `内容` ``(已有的 backtick → `<code>` 转换会兜住)
- generate_html:补 ```` ``` ```` 三反引号 → `<pre><code>` 转换,CSS 加 pre 样式(灰底,等宽字体)

### 3. macOS 双击启动

- **选定**:项目根目录放 `启动.command` 文件——一行 shell 脚本,双击即启动 GUI
- 原因:零依赖、零构建。用户可拖到 Dock 或桌面
- 已知不做:`py2app` 打真 `.app` 图标包(以后有需求再升级)
- 首次运行需给它可执行权限(`chmod +x`);会在 `setup_mac.sh` 里做

### 4. 剪贴板自动填 URL

- **选定**:GUI 初始化时读一次剪贴板;若内容是 `http(s)://` 开头的 URL,自动填入
- 原因:tkinter 内置支持,零依赖;失败(剪贴板为空/权限)静默忽略
- 已知不做:实时监听剪贴板变化(有,但会让用户困惑)

### 5. 批量保存

- **选定**:单行 URL 输入框改成多行文本框(3 行高);每行一个 URL,空行忽略
- 处理:逐个抓,日志显示 "1/3 → 2/3 → ..."
- 单个失败不影响其他;结束显示总结("成功 2 / 失败 1")
- 已知不做:并发抓(网站反爬对并发敏感,顺序更稳)

## 测试

- 链接:单元测试(纯文本链接、加粗内含链接、锚点链接被过滤)
- 代码块:单元测试(pre 块、行内 code、pre 里含加粗)
- GUI 剪贴板、批量:目测 + 一个非交互式集成测试(把 URL 列表塞进去看能否切分)
- 现有 28 个测试不许坏
- 真实文章端到端

## 明确不做(YAGNI)

图片链接可点击放大;代码语法高亮;剪贴板实时监听;并发抓取;批量结果导出为 CSV。
